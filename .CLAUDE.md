## 3. INVARIANTS (hard constraints — assert at generation time; violation = stop and report)

Core chain: `raw multimodal load → physical-coordinate alignment (thermal x / SEM tile / height-map x)
→ target extraction from height map → train/val/test split → model → metric → threshold`

- **scaler (if any) is fit on train only** — never touches val/test.
- **Track 21 is the held-out test set, fixed** — not part of any CV rotation. This is the
  paper's own recommendation (§5/§6 of `paper/2607.07965v1.pdf`): Track 21 has the worst
  height-map NaN coverage (55.5%) of the four tracks, so holding it out tests robustness to
  incomplete profilometry rather than tuning on it. Model selection / hyperparameter tuning
  uses cross-validation over Tracks 8, 10, 14 only; Track 21 is touched once, at the end.
- **SEM input features must exclude the processed track region** — SEM is context-only
  (surrounding substrate morphology); using the track itself as an input leaks the target
  (this is called out in README.md, §"Challenge task", and the paper's §2).
- **all three modalities are aligned on the same physical x window (20–100 mm)** before
  being paired into a sample — never index thermal frames / SEM tiles / height-map columns
  by raw array position without going through the physical-mm coordinate.
- **height-map NaN fraction is checked per sample window, not silently zero-filled.**
  All four tracks have substantial missing profilometry (37–56% NaN, per paper Table 2).
  A window with NaN fraction above a stated cutoff is marked invalid and excluded from
  target extraction — treating NaN as 0 silently biases the width/boundary estimate.
- **threshold has a single source** — not recomputed independently in multiple places.
- **metric has a single implementation** — shared project-wide, never copied.
- **data_hash matches**: fingerprint (e.g. checksum) of the raw Zenodo file(s) feeding a
  processed run == what's expected. Raw files aren't in git, so this is the only thing that
  catches a stale/mismatched local download.

of "got it." So do this instead.

Produce three blocks, **left blank for me to fill first**:

I write predictions first → run → **wherever they don't match is a hole in my understanding.**
Your confirmation (answers / explanation) comes only *after* I've written my predictions, not before.
